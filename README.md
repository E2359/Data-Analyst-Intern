# Data Analyst Intern Report

## Project Overview

This project is a restaurant business analysis for **Chicken GoGo**. The goal of this project is to analyze real restaurant operation data and turn it into business insights that can help improve sales tracking, labor scheduling, payroll review, and overall restaurant management.

The project includes both a professional HTML report and a Jupyter Notebook analysis.

## Files Included

| File | Description |
|---|---|
| `ChickenGoGo_Data_Analyst_Intern_Report.html` | Final HTML report with KPI summaries, charts, business insights, and recommendations. |
| `ChickenGoGo_Data_Analyst_Intern_Restaurant_Analysis.ipynb` | Jupyter Notebook containing the data cleaning, analysis process, visualizations, and business interpretation. |

## Business Questions

This project focuses on several restaurant operation questions:

1. How is the restaurant performing based on sales and payment data?
2. What do card sales, cash sales, and tips show about business activity?
3. Are labor hours and payroll records reasonable compared with restaurant operations?
4. Are employees scheduled properly based on role and business needs?
5. What possible payroll, clock-in, or scheduling issues should be reviewed?
6. How can the restaurant improve sales tracking, labor planning, and cost control?

## Key Insights

This project shows how restaurant data can be used to better understand business operations. The analysis helps identify sales trends, labor patterns, schedule coverage issues, and areas where the restaurant can improve data tracking.

Some major areas reviewed include:

- Daily sales performance
- Cash and card payment patterns
- Tip collection trends
- Employee labor hours
- Payroll consistency
- Schedule coverage
- Operational risks and improvement opportunities

## Recommendations

Based on the analysis, the restaurant should:

1. Track daily sales in a consistent format.
2. Record item-level sales for better menu analysis.
3. Compare scheduled hours with actual clock-in records.
4. Review payroll records regularly for possible errors.
5. Track labor hours by role, such as front desk and kitchen.
6. Build a weekly dashboard for sales, payroll, and scheduling.
7. Collect more detailed inventory and supplier cost data.

## Limitations

This analysis is based on the available restaurant files. The project would be stronger with additional data, such as:

- Item-level sales
- Hourly sales
- Customer count
- Inventory usage
- Waste tracking
- Delivery app reports
- Longer-term sales history

Because of these limitations, some findings should be treated as business signals rather than final conclusions.

## Project Summary

This project shows how a data analyst intern can support a restaurant by organizing business data, analyzing sales and labor patterns, creating useful KPIs, and presenting recommendations in a professional report. It demonstrates how data analysis can help small businesses make better operational decisions.
